# 价格优先级

当 PRUNplanner 计算价格时，它遵循以下优先级顺序：

- 基于星球物料 (Planet Material-based)
- 基于帝国物料 (Empire Material-based)
- 基于星球交易所 (Planet CX-based)
- 基于帝国交易所 (Empire CX-based)
- 备选方案：PP30 天全宇宙平均价 (PP30 Day Universe Price)

# 偏好类型

你可以将偏好设置为 买入 (BUY)、卖出 (SELL) 或 全部 (BOTH)：

- **买入 (BUY)** 用于物料被消耗的情况。
- **卖出 (SELL)** 用于物料被产出的情况。
- **全部 (BOTH)** 适用于上述两种情况。

如果仅设置了一种类型，而计算需要另一种类型，PRUNplanner 将检查更高层级的偏好。如果均不存在，则数值默认为 0。

# 示例

1：需要使用 LST 的价格。你尚未定义基于 Limestone 物料的偏好。系统会检查你的交易所偏好：首先是星球层级，然后是帝国层级。由于你两者都定义了，星球交易所偏好具有优先级。

2：需要使用 NS 的价格。你在帝国和星球层级都定义了基于 NS 物料的偏好。星球物料偏好将覆盖帝国物料偏好。由于存在基于物料的偏好，因此不会应用任何交易所偏好。

3：需要使用 BBH 的价格。你没有设置任何基于物料或基于交易所的偏好。因此，BBH 的价格将使用 PP30D 全宇宙指标进行计算，如果无数据可用，则回退到 0.00。
